# External integrations

All external providers are optional. The local URL/text/prompt models, local PE
inspection, RDAP heuristics, and template explanations continue to work when a
provider is disabled.

## Preflight

Run the provider-safe CLI:

```powershell
python -m scripts.check_integrations
```

Or, while the backend is running:

```text
GET /v1/integrations/status
```

Both outputs contain only readiness, missing environment-variable names, and
safe operational notes. They never return API keys, OAuth secrets, bank account
numbers, AWS resource IDs, or provider endpoints.

## Required deployer-owned configuration

| Capability | Required values | Verification |
|---|---|---|
| Gmail | `GMAIL_OAUTH_CLIENT_ID`, `GMAIL_OAUTH_CLIENT_SECRET`, redirect URI, Fernet key | Complete OAuth once, then call `/v1/integrations/gmail/status` |
| SePay | webhook HMAC secret (preferred), bank name/account/owner | Send a signed sandbox webhook for an exact pending order |
| Cloud sandbox | AMI, subnet, security group and standard AWS SDK credentials | Create and terminate one disposable test session |
| MetaDefender | `METADEFENDER_API_KEY` | Perform hash lookup; sample upload remains explicit opt-in |
| ClamAV | private `CLAMAV_HOST`/port | `/v1/ready` reports the daemon probe |
| OCR | Tesseract in `PATH` or `TESSERACT_EXECUTABLE` | `/v1/integrations/status` reports `ocr.ready=true` |
| MISP | enable flag, HTTPS base URL and API key | Run a bounded indicator lookup |
| Commercial domain/phone reputation | corresponding server-side API key | Run one non-sensitive test lookup |

Credentials cannot be generated by the application. Create them in the
provider account owned by the deployment, place them in an uncommitted `.env`,
and rerun the preflight. Never embed provider secrets in the web, desktop, or
extension bundles.
